Venv and Pip in Visual Studio Code

Introduction

Python developers often rely on venv and pip to manage project environments and dependencies. This guide explains how to use these tools effectively within Visual Studio Code to create isolated environments and manage Python packages for your projects.

What is venv?

venv (virtual environment) is a tool for creating isolated Python environments. Each environment is self-contained and includes its own installation of Python and libraries. This ensures that dependencies for one project do not interfere with another.

Key Benefits:

  • Prevents dependency conflicts between projects.
  • Enables reproducible environments for collaboration and deployment.
  • Helps maintain a clean global Python installation.

Creating a Virtual Environment in Visual Studio Code


# Create a virtual environment in your project directory
python -m venv venv

# Activate the virtual environment
# On Windows
venv\Scripts\activate

# On macOS/Linux
source venv/bin/activate

Once activated, you'll see the virtual environment's name in your terminal prompt (e.g., (venv)). In Visual Studio Code, the environment can be selected in the bottom-left corner.

Basic Commands:


# Install a package
pip install package_name

# Install specific version
pip install package_name==1.0.0

# List installed packages
pip list

# Upgrade pip itself
pip install --upgrade pip

Installing Packages in Visual Studio Code

With the virtual environment activated in Visual Studio Code, you can use the integrated terminal to install dependencies. For example, to install pandas and matplotlib, run: 


pip install pandas matplotlib

Installed packages are stored within the virtual environment, ensuring isolation from the global Python installation.

Learn more about pip.

Using requirements.txt with pip

A requirements.txt file lists all dependencies for a project, enabling reproducibility and easier collaboration.

Creating a requirements.txt File


# Generate a requirements.txt file
pip freeze > requirements.txt

Installing Dependencies from requirements.txt


# Install dependencies from requirements.txt
pip install -r requirements.txt

This approach ensures all team members or deployment environments use the same package versions.

Best Practices

  • Always use a virtual environment for each project.
  • Regularly update your requirements.txt file.
  • Keep pip and virtual environment tools updated.
  • Avoid installing unnecessary packages globally.

Conclusion

Using venv and pip together is a powerful way to manage Python projects effectively. By isolating dependencies and leveraging Visual Studio Code's environment features, you can ensure clean, maintainable, and reproducible workflows.
